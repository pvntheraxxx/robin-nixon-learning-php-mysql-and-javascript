Конвейерная обработка в WindowsPowershell

Конвейерная обработка - это построение конвейера, в котором несколько
командлетов по очереди обрабатывают одни и те же объекты или их части.
(Pipline)

Вся суть конвейерной обработки заключается в том, что выполняется некая
команда и результат её выполнения идёт не к нам на экран, как это обычно
происходит, а на вход следующей команды, а она уже выполняет свои какие-либо
действия.

Здесь необходимо, чтобы соблюдалось два условия:

A) Чтобы объект, который получает первая команда подходил 
ко второй команде и для этого есть шесть вариантов, как он может
подойти.
B) Команда, которая принимает объект в принципе поддерживала такую
возможность, чтобы её параметры поддерживали вход по конвейеру.

Командлет -Параметры

Конвейер и поток 

PS> Командлет | Where ... | Select ... | Sort ... | FT

Синтаксический анализатор (Parser)
=> Командлеты => Конвейерный обработчик (Pipeline Processor)

Шаги процесса привязки параметров:

1. Привязка всех именованных параметров
2. Привязка всех позиционных параметров 
3. Привязка из конвейера по значению с точным совпадением 
4. Привязка из конвейера по значению с конвертацией 
5. Привязка из конвейера по имени с точным совпадением 
6. Привязка из конвейера по имени с конвертацией
